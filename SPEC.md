# Git Chat UI

Git Chat UIは、Git操作をGUIで行うためのツールです。Gitのコマンドを覚える必要がなく、直感的に操作できるように設計されています。



## Overview

画面は以下のような構成になっています。

- dashboard: Gitリポジトリを選択するためのドロップダウンメニュー
- controller: Gitの操作を行うためのコントローラー画面
- config: GitやAIのトークンの設定を行うための画面

### dashboard

#### 機能

- $HOMEディレクトリにあるGitリポジトリを自動的に検出し、ドロップダウンメニューに5件表示する（それ以上はスクロールで表示）
- ドロップダウンメニューの上にInputフィールドを配置し、リポジトリ名でフィルタリングできる
- ドロップダウンメニューからリポジトリをクリックすると、そのリポジトリのGit操作画面（controller）に遷移
- 一度選択したリポジトリは、次回以降のアクセス時にドロップダウンメニューの上部に表示（Recently Used）

### controller

#### 画面構成

1. Branch List(Remote, Localのブランチを表示)
2. コミットグラフ(日付、commite ユーザー名、コミットメッセージ、コミットIDを表示)
3. git操作パネル(git add, git commit, git push、git stashをファイル単位の操作を行う)


#### Branch List機能

- RemoteとLocalのブランチを表示
- ブランチをクリックすると、そのブランチの最新のコミットにホバーが移動
- ダブルクリックすると、そのブランチに切り替え
- コミット名にスラッシュが含まれている場合は、スラッシュの前の部分をグルーピングしフォルダのように表示（例: feature/loginは、featureグループの中にloginブランチがあるように表示）
- ブランチフォルダは、クリックすると展開され、ブランチが表示される。もう一度クリックすると折りたたまれる

#### コミットグラフ機能

- コミット単位で、日付、ユーザー名、コミットメッセージ、コミットIDを表示
- コミットをクリックすると、そのコミットの変更されたファイル、差分、コミットメッセージの全文、コミットID、ユーザー名、日付を表示、
- コミットをダブルクリックすると、そのコミットにチェックアウト
- 見た目は、GitHubのコミットグラフを参考にし、コミットの順番がわかる
- コミットグラフは、ブランチの切り替えに応じて更新
- コミットグラフは、リポジトリの更新に応じてリアルタイムで更新
- コミットグラフは、スクロール可能で、過去のコミットも50件ずつ表示し、さらにスクロールするとさらに過去のコミットも読み込み表示

#### git操作パネル機能

- git add: 変更されたファイルのリストを表示し、ファイル単位でステージングできる
- git commit: コミットメッセージ、コミットタイトルを入力するフィールドと、コミットボタンを配置。コミットボタンをクリックすると、ステージングされたファイルをコミット。
- コミットタイトルの右端にフラッシュアイコンを配置し、クリックすると、コミットのファイルを見て、コミットタイトルを自動生成する機能を実装（AIによる自動生成）
- ドラッグ・アンド・ドロップで、ファイルをステージング・アンステージングできる
- スタッシュエリア(ちょっと大きめのアイコン)を用意し、その上にファイルをドラッグ・アンド・ドロップすることで、スタッシュできる
- スタッシュエリアには、スタッシュされたファイルのリストを表示

## 技術スタック

- Macのデスクトップで動作するTauriアプリケーション
- フロントエンドは、React、TypeScript、Tailwind CSSを使用して実装
- Git操作は、Node.jsのchild_processモジュールを使用して、Gitコマンドを実行
- コミットグラフの表示には、Anime.jsの最新バージョンを使用して、アニメーションを実装
- コミットタイトルの自動生成には、OpenAIのトークン or ClaudeCodeのトークン
