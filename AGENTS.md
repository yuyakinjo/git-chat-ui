# AGENTS

## インクリメンタルブランチ戦略

ユーザーと対話して、コードの変更が発生した場合、すべてのコードの変更完了後に、下記のルールに従ってgit操作してください。

1. 現在のブランチ名の末尾に「/${incremental}」を追加して、新しいブランチを作成します
  1.1 - incremental は、現在のブランチ名末尾の vN を +1 した値とする
  1.2 - vN が存在しない場合は v0 とする
  1.3 例： `feature-branch` → `feature-branch/v0`、 `feature-branch/v1`→`feature-branch/v2`
  1.4 gitでは `foo` ブランチが存在すると `foo/v0` を作成できないため、初回（v0作成時）は元ブランチを `git branch -m <旧名> <新名>` でリネームしてから作成してください。例： `git branch -m feat/dashboard feat/dashboard/v0`
2. 新しいブランチに切り替えます
3. コードの変更をコミットします
  3.1 ただしデフォルトブランチの場合は、コミットせずに5に進んでください
4. 特に指定がなければ、最初の40文字はチャットの内容を要約し、41文字以降はユーザーの入力をそのままコミットメッセージに使用してください。
5. `.incremental`フォルダを作成し、${git-sha}.jsonファイルを保存します。ファイルの内容は、以下の形式で、コミットのSHAとユーザーの入力を含むJSONオブジェクトです。
```"${git-sha}".json
{
  "git_sha": "${git-sha}",
  "user_input": "${user-input}"
}
```
6. 「コードの変更が発生したか」の判断は AI が行う
7. git diff が空の場合は、ブランチ作成・commit・json作成を行わない
8. 変更が完了したら、ユーザーにコードの変更が完了を伝えてください。
