# ブランチインクリメンタル戦略

ユーザーと対話して、コードの変更が発生した場合、すべてのコードの変更完了後に、下記のルールに従ってgit操作してください。

1. 現在のブランチ名の末尾に「/${incremental}」を追加して、新しいブランチを作成します
  1.1 例： `feature-branch` → `feature-branch/v0`、 `feature-branch/v1`→`feature-branch/v2`
2. 新しいブランチに切り替えます
3. コードの変更をコミットします(ただしデフォルトブランチの場合は、コミットせずに5に進んでください)。
4. 特に指定がなければ、最初の40文字はチャットの内容を要約し、41文字以降はユーザーの入力をそのままコミットメッセージに使用してください。
5. .incrementalフォルダを作成し、${git-sha}.jsonファイルを保存します。ファイルの内容は、以下の形式で、コミットのSHAとユーザーの入力を含むJSONオブジェクトです。
```"${git-sha}".json
{
  "git_sha": "${git-sha}",
  "user_input": "${user-input}"
}
```
1. 変更が完了したら、ユーザーに「コードの変更が完了しました」と伝えてください。
2. ユーザーが「次の変更を行う」と言った場合、ステップ1に戻ります。
